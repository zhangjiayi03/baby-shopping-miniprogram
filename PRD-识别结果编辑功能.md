## 2. 产品概述

### 2.1 背景
当前小程序已支持批量上传订单截图并自动识别商品信息，但识别结果可能不准确（如店铺名被识别为商品名），用户无法在保存前修正，保存后也无法查看详情或修改。

### 2.2 目标
- 保存前：允许用户修改识别结果
- 保存后：支持查看详情并修改

### 2.3 用户价值
- 提高数据准确性
- 减少因识别错误导致的重复录入
- 提升用户体验

## 3. 核心功能

### 3.1 功能列表

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 保存前编辑 | 识别结果展示后，保存前可修改商品名、价格、分类等 | P0 |
| 保存后查看详情 | 已保存记录可点击查看详情 | P0 |
| 保存后编辑 | 详情页可修改记录内容 | P0 |
| 编辑历史记录 | 支持编辑已保存的历史记录 | P1 |

### 3.2 功能详情

#### 3.2.1 保存前编辑（P0）

**场景描述**：用户上传图片后，系统识别出商品信息，在点击"保存"前，用户可以修改识别结果

**交互流程**：
1. 用户上传图片，系统识别完成
2. 识别结果以卡片形式展示（商品名、价格、分类、平台）
3. 用户点击卡片上的"编辑"按钮
4. 弹出编辑弹窗，可修改各项内容
5. 用户修改后点击"确认"
6. 更新展示结果，用户点击"保存"后写入数据库

**编辑字段**：
- 商品名（文本输入）
- 价格（数字输入）
- 数量（数字输入）
- 分类（选择器：喂养/洗护/服装/玩具/医疗/教育/其他）
- 平台（选择器：淘宝/京东/拼多多/抖音/美团/其他）
- 购买日期（日期选择器）

#### 3.2.2 保存后查看详情（P0）

**场景描述**：用户在列表页看到已保存的记录，点击可查看详情

**交互流程**：
1. 用户在首页/列表页看到记录
2. 点击记录卡片
3. 进入详情页，展示完整信息

**详情页展示**：
- 商品名
- 价格
- 数量
- 单价（自动计算）
- 分类
- 平台
- 购买日期
- 创建时间
- 订单截图（如有）

#### 3.2.3 保存后编辑（P0）

**场景描述**：用户在详情页可以修改记录内容

**交互流程**：
1. 用户在详情页点击"编辑"按钮
2. 进入编辑模式，可修改各项内容
3. 修改后点击"保存"
4. 更新数据库，返回详情页

## 5. 技术方案建议

### 5.1 前端方案

#### 5.1.1 编辑弹窗组件
- 使用微信小程序原生组件
- 支持表单验证
- 支持键盘快捷操作（回车确认）

#### 5.1.2 详情页
- 新建 detail 页面
- 支持编辑模式切换
- 支持图片预览

### 5.2 数据模型

```json
{
  "_id": "xxx",
  "productName": "商品名",
  "price": "99.00",

  "unitPrice": "99.00",
  "platform": "taobao",
  "categoryId": 1,
  "orderTime": "2026-04-01",
  "imageUrl": "cloud://xxx",
  "createTime": 1712000000000,
  "updateTime": 1712000000000
}
```

#### 5.2.2 新增字段
- `updateTime`: 更新时间（已有则复用）
