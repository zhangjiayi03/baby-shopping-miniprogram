# 自我反思任务说明

## 🎯 任务目标

每天凌晨 4:00 自动执行自我反思，持续改进 PM Agent 的服务质量。

---

## ⏰ 执行时间

- **周期**: 每天凌晨 4:00 (Asia/Shanghai)
- **Cron 表达式**: `0 4 * * *`
- **时区**: Asia/Shanghai
- **首次执行**: 2026-04-05 04:00 (明天凌晨)

---

## 📋 执行流程

### 1️⃣ 回顾过去 24 小时

**读取文件**:
- `memory/YYYY-MM-DD.md` (昨天和今天)
- `.learnings/LEARNINGS.md`
- `.learnings/ERRORS.md`
- `.learnings/FEATURE_REQUESTS.md`
- 最近会话历史 (sessions_list + sessions_history)

**查找信号**:
- 用户纠正 ("不对", "错了", "应该是")
- 重复出现的问题
- 任务失败或超时
- 用户表达困惑或不解

### 2️⃣ 识别问题

**问题分类**:
| 类型 | 示例 |
|------|------|
| 沟通问题 | 回复太长、太短、太正式 |
| 理解问题 | 误解需求、遗漏细节 |
| 执行问题 | 任务失败、超时、错误 |
| 知识问题 | 信息过时、不知道新功能 |

### 3️⃣ 优化档案

**更新文件**:
| 文件 | 优化内容 |
|------|----------|
| `SOUL.md` | 行为模式、沟通风格、原则 |
| `USER.md` | 用户偏好、项目上下文、注意事项 |
| `AGENTS.md` | 工作流程、自动化规则、协作方式 |
| `TOOLS.md` | 工具使用技巧、环境配置、gotchas |

**更新原则**:
- 只写具体、可执行的规则
- 避免模糊、泛泛的描述
- 用例子说明 (好 vs 坏)

### 4️⃣ 生成进化日志

**更新文件**: `memory/evolution-log.md`

**记录内容**:
```markdown
## 📅 YYYY-MM-DD

### ✅ 今天做得好的地方
- ...

### ⚠️ 发现的问题
- ...

### 🔧 已优化的配置
- 更新了 SOUL.md: ...
- 更新了 USER.md: ...

### 📈 明天的改进计划
- ...
```

### 5️⃣ 发送反思报告

**发送方式**: message 工具 (channel: kim, target: user:221732677663142)

**报告格式**:
```
🧠 每日自我反思报告 - {日期}

✅ 今天做得好的地方：
- ...

⚠️ 发现的问题：
- ...

- ...

📈 明天的改进计划：
- ...
```

## 🚨 异常处理

**如果任务失败**:
1. 检查 cron 日志：`cat /home/node/.openclaw/cron/runs/self-reflection-4am-daily-*.jsonl`
2. 查看错误信息：`lastError` 字段
3. 手动执行反思流程
4. 修复配置问题

**如果连续 3 天失败**:
- 上报用户，请求人工干预
- 检查 agent 配置、权限、依赖
