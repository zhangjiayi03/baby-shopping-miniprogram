## 一、产品概述

### 1.1 背景

萌宝账本是一款微信小程序，帮助家长记录和管理宝宝购物消费。支持拍照/截图自动识别商品信息（百度 ERNIE-4.5-Turbo-VL 多模态模型 + OCR 后备方案），提供预算管理、消费统计等功能。

### 1.2 核心价值

- 拍照即记账：OCR 自动识别商品名、价格、分类、平台
- 多宝宝管理：支持多宝宝关联，按宝宝筛选消费
- 预算管控：月度预算设置与进度追踪
- 消费洞察：分类占比、平台分布、月度趋势统计

### 1.3 技术栈

- 微信小程序原生开发
- 微信云开发（云函数、云存储、云数据库）
- 百度 ERNIE-4.5-Turbo-VL 多模态模型 + OCR API
- WXS 模块（WXML 模板函数调用）

|--------|------|---------|------|

## 四、用户使用路径

### 路径一：单张拍照记账（核心路径）

```
首页 → 点击底部Tab"记账" → 记账页
  → 点击"📷 单张上传"
  → 系统调起 wx.chooseMedia（相册/相机）
  → 选择图片 → 图片预览展示
  → 自动上传到云存储 → 调用 OCR 云函数识别
  → 识别结果卡片展示（商品名、价格、分类、平台、日期）
  ├─ 满意 → 点击"保存" → record 云函数 create → 保存成功 → 清空页面
  ├─ 不满意 → 点击"编辑" → edit-dialog 弹窗
  │   → 修改商品名/价格/数量/分类/平台/日期
  │   → 点击"确认" → 更新识别结果
  │   → 点击"保存" → 保存成功
  └─ 识别失败 → 点击"重新识别" → 重新调用 OCR
```

### 路径二：批量上传记账

```
记账页 → 点击"📸 批量上传" → 跳转批量上传页
  → 选择宝宝（picker，可选）
  → 点击"选择图片" → wx.chooseMedia（最多 9 张）
  → 图片预览区展示（带序号，可单张删除）
  ├─ 点击"预览" → wx.previewImage 全屏预览
  └─ 点击"清空" → 清除所有图片
  → 点击"开始识别 (N张)"
  → 逐张上传 + OCR 识别（显示 loading）
  → 识别结果列表展示：
  │   ✅ 识别成功 → 商品名 + 价格（两位小数）
  │   ❌ 识别失败 → 错误原因
  ├─ 单条编辑 → 点击"编辑" → edit-dialog → 确认后更新（含 priceDisplay 同步）
  ├─ 单条保存 → 点击"保存" → record 云函数 create → 状态变为"已保存 💾"
  └─ 批量保存 → 点击"批量保存 (N条)" → record 云函数 batchCreate
      → 弹窗"成功保存 N 条记录，是否查看？"
      ├─ 确定 → switchTab 到首页
      └─ 取消 → 留在当前页
```

### 路径三：查看和编辑记录

```
首页 → 记录列表
  → 点击卡片主体 → 跳转详情页
  → 详情页展示模式：
  │   商品名 / 价格（¥XX.XX）/ 数量 / 单价（¥XX.XX）
  │   宝宝 / 分类 / 平台 / 购买日期 / 创建时间
  │   订单截图（可点击 wx.previewImage 预览）
  │
  ├─ 点击"编辑" → 进入编辑模式
  │   → 修改商品名（input）/ 价格（digit input）/ 数量（number input）
  │   → 选择宝宝（picker）/ 分类（picker）/ 平台（picker）/ 日期（date picker）
  │   → 点击"保存"
  │   │   ├─ 宝宝未变更 → 直接调用 record 云函数 update
  │   │   │   → 更新成功 → 返回展示模式（priceDisplay/unitPriceDisplay 同步更新）
  │   │   └─ 宝宝变更 + 有 batchId → 弹出同步修改弹窗
  │   │       "检测到本记录来自批量上传，是否同时修改同批次其他 N 条记录的宝宝信息？"
  │   │       ├─ "仅修改当前" → record update（仅当前记录）
  │   │       └─ "同步修改全部" → record batchUpdateBaby（同批次）+ record update（当前记录）
  │   └─ 点击"取消" → 恢复原始数据，返回展示模式
  │
  └─ 点击"删除" → 确认弹窗"确定要删除这条记录吗？"
      ├─ 确定 → record 云函数 delete（含云存储图片清理）→ navigateBack 返回首页
      └─ 取消 → 不操作
```

**保存编辑时的分支逻辑**：
```
saveEdit()
  ├─ 表单验证（商品名非空 / 价格>0 / 数量>0）
  ├─ 宝宝未变更 → doSaveEdit(newBabyId, false)
  └─ 宝宝变更 + 有 batchId
      → showBatchModal = true
      → getBatchCount() 获取同批次其他记录数
      → 用户选择：
      │   "仅修改当前" → closeBatchModal → doSaveEdit(newBabyId, false)
      └   "同步修改全部" → confirmBatchUpdate → doSaveEdit(newBabyId, true)
          → batchUpdateBaby（同批次其他记录）
          → update（当前记录所有字段）
          → 更新页面展示数据（含 priceDisplay/unitPriceDisplay）
```

## 六、功能详情

### 6.1 识别结果编辑

#### 6.1.1 保存前编辑 - 单张上传（P0）

**场景**：用户在 `pages/record/record` 上传图片后，系统识别出商品信息，在点击"保存"前可修改

**交互流程**：
1. 用户上传图片，系统识别完成
2. 识别结果以卡片形式展示（商品名、价格、分类、平台）
3. 用户点击"编辑"按钮
4. 弹出 `edit-dialog` 组件，可修改各项内容
5. 用户修改后点击"确认"
6. 更新展示结果，用户点击"保存"后调用 `record` 云函数 `create` action 写入数据库

**编辑字段**：商品名（文本）、价格（digit）、数量（number）、分类（选择器）、平台（选择器）、购买日期（日期选择器）

**数据流**：
```
OCR 识别 → recognitionResult（含 priceDisplay 格式化）
→ 点击编辑 → edit-dialog 组件
→ 确认 → onEditConfirm 合并数据
→ 保存 → record 云函数 create action
```

#### 6.1.2 保存前编辑 - 批量上传（P0）

**场景**：用户在 `pages/batch-upload/batch-upload` 批量上传图片后，可逐条编辑识别结果

**交互流程**：
1. 用户选择多张图片（最多 9 张）
2. 点击"开始识别"，逐张上传并 OCR 识别
3. 识别结果列表展示，每条有"编辑"和"保存"按钮
4. 点击"编辑"弹出 `edit-dialog` 组件
5. 确认后更新该条结果数据（含 priceDisplay 同步更新）
6. 可单条保存或批量保存

**关键逻辑**：
- 编辑时记录 `editIndex` 状态，确认时从状态读取（非依赖 e.detail 传 index）
- 编辑确认后同步更新 `priceDisplay` 字段
- 批量保存使用 `record` 云函数 `batchCreate` action
- 单条保存使用 `create` action
- 识别结果包含 `imageUrl`（云存储 fileID）

#### 6.1.3 保存后查看详情（P0）

**场景**：用户在首页列表页看到已保存的记录，点击可查看详情

**详情页展示**：商品名、价格（priceDisplay）、数量、单价（unitPriceDisplay）、宝宝关联、分类、平台、购买日期、创建时间、订单截图（可点击预览）

**数据加载**：调用 `record` 云函数 `get` action，含权限校验（_openid 验证），分类/平台名称从 `app.globalData` 映射获取

#### 6.1.4 保存后编辑（P0）

**场景**：用户在详情页可以修改记录内容

**交互流程**：
1. 用户在详情页点击"编辑"按钮
2. 进入编辑模式，字段变为可编辑状态
3. 修改后点击"保存"
4. 调用 `record` 云函数 `update` action 更新数据库
5. 返回详情展示模式

**更新逻辑**：
- `update` 云函数有字段白名单：`productName, price, quantity, categoryId, platform, babyId, orderTime, imageUrl`
- 更新时自动重算 `unitPrice`（price / quantity）
- 更新时自动设置 `updateTime`

#### 6.1.5 同批次宝宝同步修改（P1）

**场景**：修改记录的宝宝关联时，如果该记录来自批量上传（有 batchId），可选择同步修改同批次其他记录

**交互流程**：
1. 用户在详情页编辑模式修改宝宝关联
2. 系统检测到 batchId 存在
3. 调用 `record` 云函数 `batchCount` action 获取同批次其他记录数
4. 弹出确认弹窗："是否同时修改同批次其他 N 条记录的宝宝信息？"
5. 用户选择：
   - "仅修改当前"：只修改当前记录
   - "同步修改全部"：调用 `batchUpdateBaby` action 更新同批次其他记录的 babyId

**关键逻辑**：
- `batchUpdateBaby` 只更新同批次其他记录（`_id: _.neq(recordId)`），不重复更新当前记录
- 当前记录的其他字段通过 `update` action 单独更新

### 6.2 首页功能

#### 6.2.1 预算卡片（budget-card 组件）

**展示内容**：本月预算金额、已花费金额、剩余金额（浮点精度修正）、预算进度条（百分比）、颜色动态变化（≤80% 绿 / 80%-100% 黄 / >100% 红）

**样式**：
```
┌─────────────────────────────┐
│ 📊 本月预算                  │
│ ████████████░░░░░  65%      │
│ 已花 ¥1,280 / 预算 ¥2,000   │

└─────────────────────────────┘
```

**组件属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| budget | Number | 月度预算金额 |
| spent | Number | 已花费金额 |

**数据来源**：
- `budget`：从 `wx.getStorageSync('monthlyBudget')` 读取，默认 2000
- `spent`：从 `stats` 云函数 `getStats` action 获取 `thisMonth.spent`

#### 6.2.2 今日花费卡片（stats-card 组件）

**展示内容**：今日花费金额、较昨日对比（↑↓百分比）、今日订单数

**样式**：
```
┌─────────────────────────────┐
│ 💰 今日花费                  │
│ ¥156.00                     │
│ 较昨日 ↑12%  (3 笔)          │
└─────────────────────────────┘
```

**组件属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| today-spent | Number | 今日花费 |
| yesterday-spent | Number | 昨日花费 |
| order-count | Number | 今日订单数 |

**环比计算逻辑**：
- 昨日花费 > 0 时：`changePercent = Math.round(((today - yesterday) / yesterday) * 100)`
- 上涨：↑ 红色 / 下降：↓ 绿色 / 持平：→ 无颜色

**数据来源**：从 `stats` 云函数 `getStats` action 获取 `today.spent`、`yesterday.spent`、`today.count`

#### 6.2.3 筛选区

| 筛选器 | 数据源 | 默认选项 |
|--------|--------|----------|
| 宝宝筛选 | `babies` 云函数 list | 全部宝宝 |
| 分类筛选 | app.globalData.categories | 全部分类 |
| 平台筛选 | app.globalData.platforms | 全部平台 |

**交互**：使用 picker 选择器组件，选中后高亮显示，筛选条件变更后自动刷新列表

#### 6.2.4 记录列表

**列表项展示**：商品名、宝宝标签、分类标签、平台标签、购买日期、价格（priceDisplay）、数量

**操作按钮**：✏️ 编辑（跳转详情页编辑模式）/ 🗑️ 删除（确认后调用 record delete）

**列表功能**：分页加载（pageSize: 20）、下拉刷新、加载更多/没有更多提示、空状态展示

**WXS 模块**：`index.wxs` 提供 `getCategoryName(categoryId)` 和 `getPlatformName(platform)` 在 WXML 模板中调用
